Docker 容器相互访问

橙子 2020-12-19 LinuxDocker

# 容器之间互相访问

由于 Docker 容器之间互相隔离，是不可以直接进行访问的（通过 localhost 访问）。需要通过以下几种方式来进行访问。

# 1. 公网访问

如果你的宿主机的 IP 为公网 IP，则可以直接通过该 IP 访问。

# 2. 自定义网络

首先创建一个新的虚拟网络，可以用它来管理容器通信。

$ docker network create my_network
# my_network 为虚拟网络名
1
2

假设有一个 MySQL 容器（名为 mysql），一个后端服务容器（名为 spring）。

我们可以把两个容器链接到上面的虚拟网络中。

$ docker network connect my_network mysql
$ docker network connect my_network spring
1
2

然后 spring 容器中寻找 mysql，将 IP 地址修改为容器名 mysql：

# 进入 spring 容器后执行
$ docker exec -it spring bash
$ ping mysql

# 项目配置文件中 mysql 连接地址修改为
jdbc:mysql://mysql:3306/***?
1
2
3
4
5
6

或者在容器启动时显式指定其网络（使用 --network 指定网络名)：

$ docker run --network my_network ****
1

# 3. 链接容器法

提示

该方法不再推荐，不过过去很长时间它都是经常使用的，也要了解会使用。

首先第一个容器需要命名（这里命名为 spring-mysql）：

$ docker run --name spring-mysql \
*** \   # 省略部分参数
-d mysql:8.0
1
2
3

然后在需要访问 mysql 的容器启动参数上添加（--link）：

$ docker run --name spring \
--link spring-mysql:mysql \
-d spring
1
2
3

这样在 spring 容器中对 mysql 服务的引用将发送到名为 spring-mysql 的容器中。

同样将配置文件修改 IP 地址为 mysql:

jdbc:mysql://mysql:3306/***?
