# Docker 简介

Docker

Docker 是一个允许用户“在任何地方构建、分发及运行任何应用”的平台。 Docker单词原指码头工人，比喻指将软件放在“集装箱内”。

# 1. Docker 的好处

某种程度上代替虚拟机（如果用户只关心应用程序而不是操作系统）

隔离环境，不影响全局环境。

打包软件、快速部署。

持续交付、迁移便捷。

# 2. 基本概念

镜像：

镜像是文件系统层和一些元数据的集合，他们合在一起，以 Docker 容器的形式运行。

容器：

容器是镜像的运行实例。可以使用同一个镜像运行多个容器。

提示

Docker 镜像是运行容器的模板，这与程序文件和运行进程的差异类似； 也可以理解为类与对象的区别。

层：

层是文件变更的集合。

仓库：

一个仓库管理服务中可以包含多个仓库（Repository）；

每个仓库可以包含多个 标签（Tag）；

每个标签对应一个镜像。

通常，一个仓库会包含同一个软件不同版本的镜像，而标签就常用于对应该软件的各个版本。 我们可以通过 <仓库名>:<标签> 的格式来指定具体是这个软件哪个版本的镜像。 如果不给出标签，将以 latest 作为默认标签。

# 镜像操作

# 1. 查找镜像

docker search [OPTIONS] TERM
1
# 从Docker Hub查找镜像名包含java的镜像
$ docker search java
1
2

列表说明：

  • NAME: 镜像仓库源的名称。
  • DESCRIPTION: 镜像的描述。
  • stars: 类似 Github 里面的 star，表示点赞、喜欢的意思。
  • OFFICIAL: 是否 docker 官方发布。
  • AUTOMATED: 自动构建。

# 2. 获取镜像

$ docker pull [选项] [Docker Registry 地址[:端口号]/]仓库名[:标签]
1

选项：

-a ：拉取所有 tagged 镜像。

Docker 镜像仓库地址：

地址的格式一般是 <域名/IP>[:端口号]。默认地址是 Docker Hub(docker.io)。

仓库名：

如之前所说，这里的仓库名是两段式名称，即 <用户名>/<软件名>。 对于 Docker Hub，如果不给出用户名，则默认为 library，也就是官方镜像。

标签：

通常为版本号，如果不指定，则默认为 latest。

# 3. 列出镜像

显示顶层镜像，列出 仓库名、标签、镜像 ID、创建时间 以及 所占用的空间：

$ docker image ls
1

显示虚悬镜像：

$ docker image ls -f dangling=true
1

虚悬镜像

由于新旧镜像同名，旧镜像名称被取消，从而出现仓库名、标签均为 <none> 的镜像。 一般来说，虚悬镜像已经失去了存在的价值，是可以随意删除的。

显示所有镜像（包含中间镜像）：

$ docker image ls -a
1

中间镜像

无标签的镜像很多都是中间层镜像，是其它镜像所依赖的镜像。 这些无标签镜像不应该删除，否则会导致上层镜像因为依赖丢失而出错。

# 4. 删除镜像

$ docker image rm [选项] <镜像1> [<镜像2> ...]
1

镜像：可以是 ID、镜像名、镜像摘要等，唯一标识即可。

# 容器操作

# 1. 运行容器

$ docker run
1

参数：

  • -t 选项让Docker分配一个伪终端（pseudo-tty）并绑定到容器的标准输入上。
  • -i 让容器的标准输入保持打开。
  • -d 以守护态运行，即让 Docker 在后台运行而不是直接把执行命令的结果输出在当前宿主机下，启动后会返回一个唯一的 id。 通过 docker container logs [container ID or NAMES] 命令查看输出信息。

# 2. 启动、终止和重启

当容器被关闭，可以将一个已经终止的容器启动运行：

$ docker container start [container ID or NAMES]
1

终止一个运行中的容器：

$ docker container stop [container ID or NAMES]
1

将一个运行态的容器终止，然后再重新启动它：

$ docker container restart [container ID or NAMES]
1

# 3. 删除容器

$ docker container rm [container ID or NAMES]
1

提示

删除未停止运行的容器需要加入 -f 参数发送 SIGKILL 信号给容器。

或者使用 stop 命令先停止运行容器。

清理掉所有处于终止状态的容器：

$ docker container prune
1

警告

执行此命令前，一定确保所需要的容器都在运行中，否则会误删。慎用

# 4. 进入容器

$ docker exec -it [container ID or NAMES] bash
1

参数说明：

  • -it：同上，
  • bash：使用 bash 界面。

警告

如果使用 docker attach 命令的话，从这个 stdin 中 exit，会导致容器的停止。

# 5. 导入导出容器

如果要导出本地某个容器，可以使用 docker export 命令导出容器快照到本地文件：

docker export [container ID or NAMES] > [压缩包名.tar]
1

使用 docker import 从容器快照文件中再导入为镜像：

cat ubuntu.tar | docker import - test/ubuntu:v1.0
# 也可以通过指定 URL 或者某个目录来导入
docker import http://example.com/exampleimage.tgz example/imagerepo
1
