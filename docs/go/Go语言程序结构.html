# 一、命名

命名规则涉及 变量、常量、函数、结构、接口、方法、包名 等的命名。

命名规则：

  • 以一个字母（Unicode）或下划线开头；
  • 可以跟任意数量的字母、数字或下划线；
  • 区分大小写（Hello 和 hello 不同）。
  • 不能使用关键字，以及尽量不使用预定义名。

关键字：

case        defer          go         map           struct
chan        else           goto       package       switch

continue    for            import     return        var

注：

Go 常使用短小的名字。如果一个名字的作用域比较大，生命周期也比较长，那么用长的名字将会更有意义。

命名规范：

  • 包名：全小写，简短，无下划线。package api
  • 文件名：下划线命名法。user_service.go
  • 结构体、变量、接口：驼峰命名法。
  • 常量：全大写，下划线分割。

# 二、声明

# 1. 声明类型

  1. var：变量
  2. const：常量
  3. type：类型
  4. func：函数实体对象

# 2. 声明级别

  1. 包级别：可在整个包对应的每个源文件中访问。
  2. 函数级别：只能在函数内部很小的范围被访问。

# 三、变量

两种声明变量方式： var 声明；简短声明。

# 1. var 声明语句

用于需要 显式 指定变量类型地方，或者因为变量稍后会被 重新赋值 而初始值无关紧要的地方。

var 变量名字 类型 = 表达式
1

省略：（二者只能省略一个）

  • 类型：根据 表达式 来推导。
  • 初始化表达式：零值初始化。

零值：

  • 数值类型 -> 0
  • 布尔类型 -> false
  • 字符串类型 -> 空字符串 ""
  • 接口或引用类型 -> nil
  • 数组或结构体等聚合类型 -> 是每个元素或字段对应该类型的零值

初始化：

  • 在 包级别 声明的变量会在 main 入口函数执行前完成初始。
  • 局部变量 将在声明语句被执行到的时候完成初始化。

# 2. 简短变量声明

用于声明和初始化 局部变量。

f, err := os.Open(name)
1

左侧可以是声明过的变量，这时为赋值功能。

# 四、赋值

左右两侧的类型必须完全匹配，nil 可以赋值给任何指针或引用类型的变量。

简单赋值：

x = 1
x = x * 10

# 五、类型

# 1. 类型声明语句

type 类型名字 底层类型
1
type Celsius float64 // 摄氏温度
type Fahrenheit float64 // 华氏温度
1
2

警告

即使这两个类型的底层类型相同，他们之间也是不兼容的。它们不可以被相互比较或混在 一个表达式运算。

# 2. 类型转换

对于每一个类型 T，都有一个对应的类型转换操作 T(x)，用于将 x 转为 T 类型（译注：如果 T 是 指针类型，可能会需要用小括弧包装 T，比如 (*int)(0) ）

var i int = 1
var j float64 = (float64(i))
