# 接口、实现分离

Java 集合类库将接口（ interface) 与 实现 (implementation) 分离。

**好处：**当在程序中一旦构建了集合就不需要知道究竟使用了哪种实现。

例如，使用接口类型存放集合的引用：

Queue expresslane = new CircularArrayQueue<>(100): 
expressLane.add(new Customer("Harry"));

2

一旦改变了想法，只需要修改调用构造器的地方的代码即可。

Queue expressLane = new LinkedListQueueo<>() ; 
expressLane.add(new Custonier('Harry"));

# Collection 接口

在 Java 类库中，集合类的基本接口是 Collection 接口。

重点讲一下迭代器方法：

public interface Collection {
    //...
    Iterator<E>  iterator();
}
1
2
3
4

iterator 方法用于返回一个实现了 Iterator 接口的对象。

**作用：**可以使用这个迭代器对象 依次访问 集合中的元素。

Iterator 接口包含 4 个方法：

public interface Iterator { 
    E next(); 
    boolean hasNext(); 
    void remove();  // 删除上次访问的对象
    default void forEachRemaining(Consumer<? super E> action); 
}

有 3 种使用方法：

1）构造迭代器对象，普通循环：

Collection c = . . .;
Iterator iter = c.iterator();
while (iter.hasNext()) {  // 如果到达了集合的末尾，next 方法将抛出一个 NoSuchElementException。
	String element = iter.next0； 
	//do something with element
}

2）使用 for each 循环，编译器会简单地将其翻译为带有迭代器的循环：

Collection c = . . .;
for (String element : c) { 
    //do something with element 
}

因为 Collection 接口扩展了 Iterable 接口。对于标准类库中的任何集合都可以使用 for each” 循环。

3）在 Java SE 8 中，可以调用 forEachRemaining 方法并提供一个 lambda 表达式（它会处理一个元素）。

iterator.forEachRemaining(element -> do something with element);

这行代码将对迭代器的每一个元素调用这个 lambda 表达式，直到再没有元素为止。

# Collection 接口常见的方法

Collection 接口声明了很多有用的方法，所有的实现类都必须提供这些方法。下面列举一些常见的：

# iterator

格式： Iterator < E> iterator()。

功能：返回一个用于访问集合中每个元素的迭代器。

# size

格式： int size()。

功能：返回当前存储在集合中的元素个数。

# isEmpty

格式：boolean isEmpty()

功能：如果集合中没有元素， 返回 true。

# contains

格式：• boolean contains(Object obj)

功能：如果集合中包含了一个与 obj 相等的对象， 返回 true。

# containsAll

格式： boolean containsAll(Collection<?> other)

功能：如果这个集合包含 other 集合中的所有元素， 返回 true。

# add

格式：boolean add(Object element)

功能：将一个元素添加到集合中。如果由于这个调用改变了集合，返回 true。

# addAll

格式： boolean addAll(Collection<? extends E> other) 功能：将 other 集合中的所有元素添加到这个集合。如果由于这个调用改变了集合，返回 true。

# remove

格式： boolean remove(Object obj)。 功能：从这个集合中删除等于 obj 的对象。如果有匹配的对象被删除， 返回 true。

# removeAll

格式： boolean removeAll(Collection<?> other) 功能：从这个集合中删除 other 集合中存在的所有元素。如果由于这个调用改变了集合，返回 true。

# removelf

格式： default boolean removelf(Predicate<? super E> filter)

功能：从这个集合删除 filter 返回 true 的所有元素。如果由于这个调用改变了集合， 则返回 true。

# clear

格式： void clear()

功能：从这个集合中删除所有的元素。

# retainAll

格式： boolean retainAll(Collection<?> other)

功能：从这个集合中删除所有与 other 集合中的元素不同的元素。如果由于这个调用改变了 集合， 返回 true。

# toArray

格式：Object[] toArray()

功能：返回这个集合的对象数组。

格式： <T> T[] toArray(T[] arrayToFi11)

功能：返回这个集合的对象数组。如果 arrayToFill 足够大， 就将集合中的元素填入这个数组 中。剩余空间填补 null ; 否则， 分配一个新数组， 其成员类型与 arrayToFill 的成员类 型相同， 其长度等于集合的大小， 并填充集合元素。
