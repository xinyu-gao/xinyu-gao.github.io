橙子 2019-11-11 Java

# AOP 的引入

我们在做日志处理的时候，我们会在每个模块中添加日志或者权限处理。但是一般这些代码都是相同的。

方法：为了实现代码复用，我们把这些代码分离成新的方法。

缺点：我们需要手动插入这些方法，也就是说，这两个方法是强耦合的。修改时就需要一个一个去修改。

改进： 动态代理，指定位置执行对应流程。这样可以将一些横向的功能抽离出来，形成独立模块，然后在指定位置插入这些功能。这样的思想被称为面向切面编程，即 AOP 。

# AOP 的概念

# 横切关注点

AOP 把一个业务流程分为几部分，权限检查、业务处理、日志记录，每个部分单独处理，然后把它们组织装成完整的业务流，每部分别称为切面或关注点。

# 切面

对横切关注点的抽象。

# 连接点

被拦截到的方法、字段（通过 setter、getter 方法得到）、构造器（本质也是方法）。

# 切入点

对连接点进行拦截的定义。（不一定每个连接点都进行操作）。

# 通知

拦截掉连接点之后要执行的代码，分为前置、后置、异常、最终、环绕通知 5 类。

# 目标对象

代理的目标对象，即动态代理的 target，AOP 接口作用的对象。

# 织入

切面和目标对象都是独立的，他们之间不耦合，将切面放到目标对象中的操作就叫做织入。

（Spring 中 aop.framework.ProxyFactory 用作织入器）。

# 引入

为类动态地添加方法和字段（不改代码）。

# AOP 实现

有三种实现方式：基于代理的 AOP（经典）、AspectJ 基于 XML 的配置、AspectJ 基于注解的配置。这里主要介绍 基于代理的 AOP。

# 主要接口

接口 作用 参数
MethodBeforeAdvice 在执行指定方法前调用 被调用的方法、执行时的参数、被拦截的 Bean
MethodReturningAdvice 执行完指定方法返回后调用 方法返回值、被调用的方法、执行的参数，被拦截的 Bean
ThrowsAdvice 指定方法抛出异常时调用 不需实现任何方法，实现类中定义了两个方法public void afterThrowing(Exception ex)；public void afterThrowing(Method method, Object[] args, Object target, Exception ex)

# 创建 Service 接口

推荐面向接口编程，创建一个接口，这里示例为 IAOPServices（Java 的接口名称规范为以大写字母”I“开头）。

public interface IAOPServices
{
    public String withAopMethod() throws Exception; //使用拦截器拦截的方法
    public String withNoAopMethod() throws Exception; //不会被拦截器拦截的方法
}

# 实现 Service 接口

使用 AOPServicesImpl 类中实现了上述接口，定义了 description 属性，以及 setter、getter 方法。

public class AOPServicesImpl implements IAOPServices
{
    private String description;

    public String getDescription()
    {
        return description;
    }

    public void setDescription(String description)
    {
        this.description = description;
    }

    @Override
    public String withAopMethod() throws Exception
    {
        System.out.println("AOP运行发方法：withAopMethod");
        if (description.trim().length() == 0)
        {
            throw new Exception("dexcription 的属性不能为空");
        }
        return description;
    }

    @Override
    public String withNoAopMethod() throws Exception
    {
        System.out.println("无AOP运行发方法：withNoAopMethod");
        return description;
    }
}

# 定义拦截器方法

然后定义 AOP 拦截器的方法了

public class AOPIntercepter implements AfterReturningAdvice, MethodBeforeAdvice, ThrowsAdvice
{
    @Override
    public void afterReturning(Object o, Method method, Object[] objects, Object o1) throws Throwable
    {
        System.out.println("方法" + method.getName() + "运行结束，返回值为：" + o);
    }

    @Override
    public void before(Method method, Object[] objects, Object o) throws Throwable
    {
        System.out.println("执行MethodBeforeAdvice，即将执行的方法：" + method.getName());
        if (o instanceof AOPIntercepter)
        {
            String description = ((AOPServicesImpl) o).getDescription();
            if (description == null)
            {
                throw new NullPointerException("description 属性不能为 null");
            }
        }
    }

    public void afterThrowing(Exception ex)
    {
        System.out.println("抛出了异常：" + ex.getMessage());
    }

    public void afterThrowing(Method method, Object[] args, Object target, Exception ex)
    {
        System.out.println("方法" + method.getName() + "抛出了异常：" + ex.getMessage());
    }
}

# 对象注入

1）借助 Spring 中的代理类将自定义拦截器 （这里是 AOPInterceptor）注入 NameMatchMethodPointcutAdvisor 类的 advice 属性中。##### 2）将定义好的 NameMatchMethodPointcutAdvisor 对象注入 ProxyFactoryBean 中。

2）将定义好的 NameMatchMethodPointcutAdvisor 对象注入 ProxyFactoryBean 中。

<bean id="aopInterceptor" class="org.springframework.aop.support.NameMatchMethodPointcutAdvisor">
	<property name="advice">
		<bean class="com.***.AOPIntercepter"></bean>
    </property>
    <property name="mappedName" value="withAopMethod"></property>
</bean>

 <bean id="aopService" class="org.springframework.aop.framework.ProxyFactoryBean">
     <property name="interceptorNames">
         <list>
             <value>aopInterceptor</value>
         </list>
     </property>
     <property name="target">
          <bean class="com.***.impl.AOPServicesImpl">
               <property name="description" value="basicAOP"></property>
          </bean>
      </property>
</bean>

# 获取对象：

从 Spring 容器中获取 IAopServices 对象，并分别执行两种发方法，一个添加拦截器，一个不添加拦截器。

public class AopTest
{
    public static void main(String[] args) throws Exception
    {
        ApplicationContext context = new ClassPathXmlApplicationContext(new String[]{"beans.xml"});
        BeanFactory factory = context;
        IAOPServices services = (IAOPServices)context.getBean("aopService");
        services.withAopMethod();
        services.withNoAopMethod();
    }
}
