# 简介

Lambda表达式

简洁的可传递匿名函数，没有名称，但是又参数列表、函数主体、返回类型、可能还有异常列表。

Java 8 新增

# 1. 工程化思想

相同或相似的代码，我们希望能够复用，而不是复制粘贴生成另一个代码。

一个类或者一个方法只做一件事（单一职责原则）。

复制粘贴的危害

做出了一个更新或修正，却忘了另一个

# 2. 行为参数化

让方法接受多种行为（策略）作为参数，并在内部使用，来完成不同的行为。

// 选出绿色苹果集合
public static List<apple> filterGreenApple(List<Apple> inventory) {
    List<Apple> result = new ArrayList<>();
    for (Apple apple : inventory) {
        if (GREEN.equals(apple.getColor())) {
            result.add(apple);
        }
    }
    return result;
}

这里的行为：选择绿色苹果。

我们应该把这个行为提取出来，因为我们很有可能在后面的开发中遇到更改为红色苹果的需求，或者其他过滤重量等需求。

再重新写一个类似的类会很麻烦，而且很多人会选择复制粘贴，上面说过了，很可能修改的不完整，导致代码出现 BUG（这里只有一个地方要修改，如果多处要修改的话，根据“墨菲定律”，在一个团队中是一定会有人忘记修改某处代码的）。

并且 filterGreenApple 做了两件事，

  • 过滤苹果。
  • 选择过滤标准。

这都说明我们应该把过滤标准（即行为）提取出来，作为参数。

修改代码如下（所有代码不一定在一个文件中，仅仅只是示例）：

// 定义一个接口对选择标准进行建模
public interface ApplePredicate{
    boolean test(Apple apple);
}
// 过滤绿色苹果实现类
public class AppleGreenColorPredicate implements ApplePredicate({
    public boolean test(Apple apple) {
        return GREEN.equals(apple.getColor());
    }
}
// 过滤红色苹果实现类
public class AppleRedColorPredicate implements ApplePredicate{
    public boolean test(Apple apple) {
        return RED.equals(apple.getColor());
    }
}
// 过滤苹果重量实现类
public class AppleHeavyPredicate implements ApplePredicate{
    public boolean test(Apple apple) {
        return apple.getWeight() > 150;
    }
}
public static List<apple> filterApple(List<Apple> inventory, ApplePredicate p) {
    List<Apple> result = new ArrayList<>();
    for (Apple apple : inventory) {
        if (p.test(apple)) { // 封装了过滤的条件
            result.add(apple);
        }
    }
    return result;
}
//更具不同需求，传入不同参数
filerApple(inventory, AppleRedColorPredicate);
filerApple(inventory, AppleGreenColorPredicate);
filerApple(inventory, AppleHeavyPredicate);

# 3. 匿名类

和 Java 中的局部类差不多，但是没有名字。

允许同时声明并实例化一个类（随用随建），将接口的实现类延迟到调用时声明和实例。

filerApple(inventory, new ApplePredicate(){
	public boolean test(Apple apple) {
        return RED.equals(apple.getColor());
    }
});
filerApple(inventory, new ApplePredicate(){
	public boolean test(Apple apple) {
        return GREEN.equals(apple.getColor());
    }
});

缺点：人们不要愿意用那些很麻烦的概念或功能，当你把新的行为传递给方法的时候，你不得不声明好几个实现某一个接口的类，然后实例化它，而且这个对象可能只提到一次。

# 4. Lambda 表达式

两个概念：

一等值/二等值

一等值为原始值（int 等）或者对象（使用 new 构建的）。二等值为其他有助于表示值的结构，程序运行期间不能传递的值，如方法和类。

函数化编程

把函数作为一等值传递。

并且执行时在元素之间无互动。（无可变的共享对象）

用来代替匿名类的解决方案，写法更简单。

例如：s

filerApple(inventory, (Apple apple) -> GREEN.equals(apple.getColor()));
filerApple(inventory, (Apple apple) -> RED.equals(apple.getColor()));
filerApple(inventory, (Apple apple) -> apple.getWeight() > 150);

# 使用

使用场景

当方法体长度多于几行，且行为不是一目了然的话，应该用具体的方法，而不是 Lambda。

# 1. 函数式接口

Lambda 表达式需要依赖函数式接口。

函数式接口：

  1. 只定义一个抽象方法的接口。
  2. 默认方法、Object方法不算在内。
  3. 可以用 @FunctionalInterface 注解来标注。

关于 @FunctionalInterface 注解：

  1. 便于生成更准确详细的文档。
  2. 编译器会检查是否正确，把错误提前到编译期，减少修改代价（软件测试思想：错误越晚发现，付出的修改代价越高）。

# 2. Lambda 表达式

(int s1, int s2) -> { return s1 - s2; }
1

注意事项：

  • 参数类型可以省略，编译器可以自动推断。

    (s1, s2) -> { return s1 - s2; }
    1

  • 当只有一个参数时，( ) 可以省略。

    s1 -> { return s1.length(); }
    1

  • 可以没有参数。{ } 也可以为空。

    () -> {}
    1

  • 当大于一个语句时必须选择显示返回，且 { } 和 return 不可省略。

    (s1, s2) -> {
	s3 = s1.subString(1, 2) + s2.subString(1, 2);
	s4 = s1.subString(3, 4) + s2.subString(3, 4);
	return s3 + s4; 
}

  • 只有一个语句时，才可以使用隐式语句（包括 void）。

    (s1, s2) -> s1 - s2;

  • { } 中返回 void 时，可以省略 { }。

    s -> System.out.println(s);

# 3. 函数描述符

函数式接口的抽象方法的签名基本上即使 Lambda 表达式的签名，成为函数描述符。

表示法：例如： () -> {} 代表着：参数列表为空、返回值 void。（能看懂就行）

在使用时，描述符必须与抽象方法签名一致（类似方法调用的标准）。
