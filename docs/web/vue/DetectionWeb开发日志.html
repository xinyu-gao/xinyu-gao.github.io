# 项目简述

本项目使用 vue-cli （基于 Node.js 的开发框架）开发，UI 组件使用的 element-ui。

本项目为前后端分离项目，和后端（Spring Boot）交互主要有两种方式：

  • HTTP 方式：采用 JSON 格式。
  • Websocket： 字符串格式，需要自行解析为 JSON 格式。

开发工具推荐使用 Visual Studio Code 或者 WebStorm。

# 搭配环境

# 1. Node.js 安装：

下载链接：https://nodejs.org/dist。

最好选择 v14.15.1 下载安装。

安装完成后，打开控制台，输入 node -v和 npm -v来测试是否安装好。

# 2. NVM 安装使用（选）

如果有多个 node.js 版本，可以使用 nvm 来控制版本。

下载链接：https://github.com/coreybutler/nvm-windows/release。

用命令如下：

nvm install 14.15.1  # 安装对应的版本
nvm use 14.15.1  # 引入使用对应的版本
nvm uninstall 14.15.1  # 卸载对应的版本

# 框架解释

# 1. 修改项目端口号

‌在下图两个文件中修改端口号，分别为开发环境和生产环境的端口号：

‌文件内容示例如下：

ENV = 'production'
port = 9528
1
2

或者直接修改文件vue.config.js中的 port的值：

const port = process.env.port || 9527 // dev port‌
1

# 2. 修改后端地址

修改 HTTP 请求地址，修改文件 vue.config.js 的 ip 字段：

const ip = 'http://127.0.0.1:12345' // 后端地址
1

修改 WebSocket 请求地址，修改文件 vue.config.js 的 wsAddress 字段：

const wsAddress = 'ws://127.0.0.1:12345/ws' // websocket 连接地址
1

# 2. 跨域问题

当 IP 地址和 端口号 中有一个不同时，就形成了跨域。

在文件vue.config.js的devServer字段下写入：

proxy: {
  '/api': {
    target: ip, // 后端请求地址
    ws: true,
    changeOrigin: true,
    pathRewrite: {
      '^/api': '/'
    }
  }
}
